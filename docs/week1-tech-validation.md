## 3. Prompt 优化方案

### v1 → v2 优化点

| 优化项 | v1 问题 | v2 改进 |
|--------|---------|---------|
| 输出格式 | 自由文本，需手动解析 | 严格 JSON 格式，含示例 |
| 元件识别 | 仅列名称 | 含 ref/type/value/pins 结构化 |
| 双语支持 | 仅中文 | 中英双语（"中文 / English"） |
| 错误检测 | 无 | 含 severity 级别 + 修复建议 |
| BOM 集成 | 单独调用 | full_analysis 一次返回全部 |

### 当前 Prompt 策略
1. **快速分析** (`_build_analysis_prompt`): 4 个核心维度（元件/拓扑/功能/关键节点）
2. **完整分析** (`_build_full_analysis_prompt`): 6 个维度（+BOM+错误检测），中英双语

### 关键设计决策
- 强制 JSON 输出 + 示例模板 → 解析成功率高
- 中英双语格式 → 面向国际用户
- 错误检测集成到 prompt → 减少 API 调用次数

|------|------|------|
| 元件识别 | ⭐⭐⭐⭐ | 常见元件（R/C/L/IC）识别准确 |

### 已知局限
1. 手绘电路图识别率低
2. 高密度 PCB 原理图（100+ 元件）可能遗漏
3. 非标准符号（如日标/欧标）需额外适配
4. PDF 多页文档仅分析首页
